# ClashHalo 局域网网关中枢 (旁路由) 配置指南

通过 ClashHalo，您可以非常轻松地将您的 Mac 变成局域网中的核心代理网关，从而为家庭网络中的其他设备（如手机、电视、平板、游戏主机等）提供无死角的网络代理能力。

ClashHalo 提供了一键开启的“网关中枢”能力，彻底简化了以往复杂的软路由配置。

---

## 一键开启局域网网关（旁路由模式）

这是终极解决方案（Transparent Gateway），Mac 将直接作为局域网其他设备的默认路由器与 DNS 服务器。其他设备完全无感，无需懂任何代理知识，所有 App、智能家电、游戏机都完美支持。

### 操作步骤

**1. 在 ClashHalo 中开启网关**
- 打开 ClashHalo 的「网络」页面。
- 找到并打开 **“局域网网关 (旁路由)”** 开关。
- *(如果你尚未开启 TUN，程序会自动引导并请求管理员授权。因为网关模式需要开启底层 IP 转发以及接管局域网 53 端口 DNS)*。
- 开启成功后，下方会自动显示 **「已接入设备」** 监控卡片。

**2. 获取 Mac 局域网 IP**
- 按住 `Option` 键的同时点击 Mac 顶部菜单栏的 Wi-Fi 图标，记录下当前的 **IP 地址**（通常是 `192.168.x.x`）。

**3. 配置需要代理的设备（手机/电视/游戏机）**
- 确保要代理的设备与 Mac 连接在**同一个路由器下**。
- 进入该设备的 Wi-Fi 或有线网络设置。
- 将 IPv4 获取方式从“自动获取(DHCP)”改为 **手动 (Static/Manual)**。
- **IP 地址**：填写一个和当前网段相同但不冲突的地址（例如 `192.168.1.150`）。
- **子网掩码**：通常为 `255.255.255.0`。
- **路由器 (Router / Gateway)**：填写 **Mac 的局域网 IP**。
- **DNS 服务器**：填写 **Mac 的局域网 IP**。
- 保存并重新连接网络即可。

此时，只要在设备上产生网络流量，你就能立刻在 Mac 上的 **「已接入设备」** 卡片里看到该设备的实时速率和耗费的总流量。

## 进阶玩法：自动接管全屋网络

如果你希望全家的设备**什么都不用设置**，连上 Wi-Fi 就自动走 Mac 的代理，你可以修改主路由器的 DHCP 分发规则。

### 操作步骤
1. **为 Mac 绑定固定 IP**：登录你家的主路由器后台，在 DHCP 设置中把 Mac 的 MAC 地址和当前 IP 绑定死，防止 IP 变动导致全家断网。
2. **修改主路由 DHCP 规则**：
   - 将 **“默认网关 (Default Gateway)”** 的地址，修改为你 Mac 的 IP。
   - 将 **“首选 DNS (Primary DNS)”** 的地址，也修改为你 Mac 的 IP。
3. 保存设置，重启路由器。家里的设备重新连上 Wi-Fi 时就会自动接入 ClashHalo 网关。

### ⚠️ 强烈警告：单点故障
如果修改了主路由器，意味着你的 Mac 掌管了全家的网络命脉。
一旦你的 Mac **休眠、关机、重启**，或者不小心**退出了 ClashHalo**，你家里的**所有设备将立刻断网**。
- **必须防休眠**：在 Mac 的「系统设置 -> 显示器 -> 高级」中，开启 **“当显示器关闭时，防止自动进入睡眠状态”**。或使用 Amphetamine 等防休眠软件。
- **推荐拓扑**：作为旁路由的 Mac，强烈建议使用 2.5G 或千兆**网线**直连主路由器或交换机，避免产生多倍 Wi-Fi 损耗。

## 备用方案：轻量级 HTTP 代理

如果你不想修改底层网关，只需要在某台设备上的某个浏览器里走代理，你可以使用轻量级的局域网 HTTP 代理。

1. 在 ClashHalo「网络」页开启 **“局域网共享”**，记录代理端口（默认为混合端口 `7890`）。
2. 在手机或另一台电脑的 Wi-Fi 设置中，找到 **“配置代理”** (HTTP Proxy)，改为 **手动 (Manual)**。
3. 填入 Mac 的局域网 IP 和端口 `7890`。

- **优点**：配置简单，随用随设，不会影响整个局域网拓扑。
- **缺点**：不彻底。部分系统组件、特殊 App 或游戏主机不支持 HTTP 代理，导致这部分流量会直连暴露。
