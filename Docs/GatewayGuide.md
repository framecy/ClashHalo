# ClashHalo 局域网网关中枢配置指南

ClashHalo 的“局域网网关”会把 Mac 变成局域网内的代理出口和 DNS 接管点。适合旁路由、家庭设备统一接管和临时共享网络这类场景。

## 前置条件

- Mac 已连接到局域网
- 已安装并可正常启动 ClashHalo
- 如需 TUN / 网关能力，允许 Helper 安装和系统权限授权

## 开启方式

1. 打开 ClashHalo 的「网络」页面。
2. 开启 **局域网网关 (旁路由)**。
3. 按提示完成 TUN / Helper 授权。
4. 开启后，页面会显示已接入设备与实时流量。

## 配置接入设备

在需要接管的设备上，把网络参数改成手动：

- **IP 地址**：同网段内未占用地址，例如 `192.168.1.150`
- **子网掩码**：通常是 `255.255.255.0`
- **网关 / Router**：填写 Mac 的局域网 IP
- **DNS**：填写 Mac 的局域网 IP

保存后重新连接网络，设备流量就会经过 ClashHalo。

## 家庭路由器方案

如果想让全屋设备自动接管，可以在主路由器里把 DHCP 的默认网关和 DNS 指向 Mac。

1. 给 Mac 绑定固定局域网 IP。
2. 把路由器 DHCP 的默认网关改成这个 IP。
3. 把首选 DNS 也改成这个 IP。
4. 保存后重启路由器。

这种方案的代价是：Mac 一旦休眠、关机或退出 ClashHalo，所有接入设备都会失去网络。

## 备用方案

如果你只想让单台设备的浏览器走代理，可以只开启 **局域网共享**，然后在设备的代理设置里填：

- **代理地址**：Mac 的局域网 IP
- **端口**：`7890`

这个方式只覆盖显式配置了 HTTP 代理的流量，不适合需要透明接管的设备。

## 常见问题

- **设备连不上网**：先确认 Mac 的局域网 IP 没变，网关和 DNS 都填的是同一个地址。
- **开启后没流量**：检查 ClashHalo 是否已启动内核，以及网关开关是否仍处于开启状态。
- **休眠后失效**：不要让 Mac 自动睡眠，或改用更稳定的常开设备。
